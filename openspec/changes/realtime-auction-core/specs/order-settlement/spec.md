## ADDED Requirements

### Requirement: 成交后生成最小订单
系统 MUST 在竞拍成交后生成最小订单记录，记录竞拍、商品、买家、成交价和订单状态。

#### Scenario: 封顶价成交生成订单
- **WHEN** 有效出价达到封顶价并使竞拍状态变更为 `SOLD`
- **THEN** 系统 MUST 为最高出价用户生成一条订单记录，并关联对应竞拍和商品

#### Scenario: 自然结束成交生成订单
- **WHEN** 竞拍到达结束时间且存在至少一笔有效出价
- **THEN** 系统 MUST 为最高出价用户生成一条订单记录，并记录最终成交价

#### Scenario: 无有效出价不生成订单
- **WHEN** 竞拍到达结束时间且不存在有效出价
- **THEN** 系统 MUST 将竞拍状态变更为 `ENDED`，且不得生成订单

### Requirement: 订单生成必须幂等
系统 MUST 保证同一竞拍成交结果最多生成一条有效订单。

#### Scenario: 重复结算不重复生成订单
- **WHEN** 系统因定时任务重试、接口重试或并发流程多次处理同一竞拍成交
- **THEN** 系统 MUST 返回已有订单或保持订单不变，不得创建重复订单

### Requirement: 用户和商家可以查看成交结果
系统 MUST 提供成交结果查询能力，让用户和商家查看竞拍最终状态和订单摘要。

#### Scenario: 查询已成交竞拍
- **WHEN** 用户或商家查询处于 `SOLD` 状态的竞拍结果
- **THEN** 系统 MUST 返回成交用户、成交价、成交时间、订单状态和商品摘要

#### Scenario: 查询未成交竞拍
- **WHEN** 用户或商家查询处于 `ENDED` 或 `CANCELLED` 状态且无订单的竞拍结果
- **THEN** 系统 MUST 返回最终竞拍状态，并明确表示未生成订单
